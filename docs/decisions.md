# 需求取捨決策

## 我決定 v1 優先服務的使用者

- [x] 行動者

原因：
- 行動者最直接受「已確認 / 未確認 / 不能直接行動」的差異影響。
- 當狀態清楚時，行動者本身會更安全，避免在不清楚當地需求與狀況的情況下直接行動。

## 這個使用者最想完成什麼

- 先判斷哪些資訊目前可以採取行動。
- 先避免把未確認資訊誤當成已確認事實。

## 這個使用者最怕什麼錯誤

- 依據不確定資訊做出錯誤行動。
- 到現場才發現任務已經失效或與實際不符。

## 我今天暫時不處理的需求

- 不在 v1 內建立完整任務派發流程。
- 不把所有資料都自動整理成正式任務，避免把未確認資訊偽裝成已確認。

## 我不採用的 AI 建議

- AI 建議：把看起來完整的資料直接整理成「可行動任務」。
  - 不採用原因：這會把未確認資訊包裝成已確認事實，違反目前的安全原則。

## 必須保留人工確認的判斷

- 地點是否足夠精確。
- 來源是否足以作為行動依據。
- 資訊是否已過期、是否有衝突、是否需要現場確認。

## 今天明確不做什麼

- 不新增真實災情資料或外部 API。
- 不把原始資訊改寫成正式整理後資料。

## 我仍不確定的地方

- v1 要不要把「不能直接行動」的原因做成更強的視覺警示。
- 是否需要在畫面上增加一個更清楚的「建議下一步」區塊。
