# 使用者訪談摘要

## 這次訪談帶來的關鍵發現

- 使用者最在意的不是「資料有沒有被整理好」，而是「這筆資訊現在能不能安全地採取下一步」。
- 三種 persona 都對「看起來完整但沒有被確認」的資訊非常敏感。
- 目前最需要避免的，是讓未確認資訊看起來像已確認事實。

## 主要需求傾向

- 回報者希望能快速送出資訊，並且不被系統誤解成親眼確認。
- 資訊整理者希望保留原文、推測與待確認狀態之間的差異。
- 行動者希望能一眼看到哪些資訊可以行動、哪些不能直接行動。

## 對 v1 的啟示

v1 應該優先把「狀態分層」做得更清楚，尤其是：

- 原始資訊
- 待確認 / 未確認
- 不能直接行動
- 需要人工確認

這些層級應該在畫面上更容易被辨識，而不是只靠文字說明。
